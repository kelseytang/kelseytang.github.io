UX Writing Challenges

探索介面文字的撰寫和呈現方式

內容大綱：

  • 推播訊息
  • 錯誤訊息
  • 推銷內容

關於這個作品

  1. 所有的情境均來自 Daily UX Writing
  2. The original challenge was to complete one scenario in one hour, but I extended it to 2 to 3 hours as I wanted the writing and the UI component to be coherent and fit the context

推播訊息

情境 #01

一名旅客正在機場等候回程班機，但是航班因天氣不佳而突然取消。

挑戰

為航空公司的APP寫一段訊息，通知旅客航班取消以及他們需要採取的行動。

字數限制

標題: 30 個字元
主文: 45 個字元
按鈕: 25 個字元

AAPP發出推播訊息通知旅客狀況，並請他們打開APP瀏覽詳情。
B使用「inline notification」元件，把訊息釘在主頁上方以便旅客重複閱讀。

溝通策略

  1. 訊息著重在「發生什麼」和「該做什麼」
  2. 避免承諾，因為天氣狀況不可控
  3. 確保旅客可以收到訊息
  4. 使用者可能關閉手機的「推播訊息」，因此在APP內使用「inline notification」作為替代方案

文字的語氣

情境 #02

使用者坐進車內，先把手機聯接到車子，然後開車去上班。

挑戰

通知使用者附近城鎮有火災發生而導致道路封閉，火災的影響有待確認。若火勢擴大逼近，則有立即的危險。

字數限制

標題: 30個字元
內文: 45個字元

溝通策略

  1. 標題：告知火災發生的路段名稱
  2. 內文：火勢變化難以預測，因此內文不提供行動指引，只會提到火災地點和可能的影響
  3. 出於安全駕駛的考量，訊息內容應避免引導使用者操作手機，例如：開啟APP

訊息語調

情境 #03

A sports fan is at a wedding while their favorite team is playing against their arch-rivals. Their team scores.

挑戰

Let the sports fan know about the latest play, the current score, and the key players.

字數限制

標題: 30個字元
內文: 45個字元

AThe headline only includes the latest match scores; to help users track the match's progress,
include a timeline of the match in the body sectionB.

溝通策略

  1. Users can opt-in for various sports notifications at a glance, adding emoji icons in the headline for easy identification of different sports
  2. Use common sports terminology
  3. Include the timeline of the match for useers to track the progress

情境 #04

A trucker, app-tracking his route, schedule, fuel, and deliveries, faces a dilemma: 6 stops remain before lunch and fuel, but traffic delays him. He must decide between pushing on with low fuel risk or refueling and eating now, finishing later.

挑戰

Write a push notification alerting him of this dilemma and options.

字數限制

標題: 30 個字元
主文: 45 個字元
按鈕: 25 個字元

↑ Display the the "Distant to Empty" in the headline to make the information glanceable

溝通策略

  1. The goal is to ensure the user's safety
  2. Utilize the emoji icon "⛽️" to make the notification easy to distinguish
  3. "Distant to Empty" and "what to AVOID" are the key items to communicate

錯誤訊息

情境 #01

The user is trying to rent a car using an application but the credit card on file has expired.

挑戰

Write them an error message so that they can correct the problem.

字數限制

標題: 30 個字元
主文: 45 個字元

AThe headline tells the user to update the credit card, then the button Balso uses the verb "Update"
to reinforce the call to action.

溝通策略

  1. Since this will impact the rental service flow, use the inline notification componment, and display it until the issue is fixed.
  2. The message should explain why the user need to take an action
  3. The app should not force the user to immediately fix the credit card to use the app, because the user may need time to get a valid credit card

訊息語調

情境 #02

The user works in graphic design. While critiquing a design in a mobile app, their phone abruptly turns off. When they restart the phone, they reopen the app.

挑戰

Write a message that the user will read immediately upon opening the app. What do they need to know? What steps (if any) do they need to take to recover their content? What if they can't recover the content?

字數限制

標題: 40 個字元
主文: 120 個字元
按鈕: 20 個字元

When users reopen the app, they are prompted to take action: resume editing or delete the draft.

溝通策略

I assume the app has temporarily stored the draft file, so the message provides 2 actions for the user:

  1. Resume the editing
  2. Delete the draft to free the device's memory

情境 #03

The user entered the wrong email address to sign in to their account.

挑戰

Tell the user to enter the right email.

字數限制

40 characters max

AThere is a chance that the user hasn't yet created an account, so include a "sign-up" button.

溝通策略

  1. It's difficult to determine if a user has entered the wrong email, so the message only states that there is no match for the provided email.
  2. In case the user doesn't have an account, include a "sign-up" button in the message

情境 #04

A user is creating an account. When they come to the step where they are asked to enter their name, they get an error message. A fraud detection software thinks their name is fake—but it’s wrong 5% of the time.

挑戰

Write an error message that prompts them to fix the error without shaming them for having a fake-sounding name.

字數限制

45 個字元以內

AThe UI label tells the user to input the name on the ID card.
BWhen the error occurs,the app reminds the uers to use the name on the ID card, and explain what could cause errors

溝通策略

  1. Avoid blaming users, explain what could cause this error
  2. If the name doesn't contains digits or symbol, the app should not block users as the system is not 100% correct
  3. A simple way to clarify the term "real name" is to instruct the user to use the name appeared on their ID card

訊息語調

情境 #05

A user is shopping using a price comparison app that boasts “real-time” pricing on items. As they are checking the price of an item, something goes wrong. The problem is unknown.

挑戰

Write a message that informs the user that they cannot access the app right now. You cannot specify "why" the app doesn't work, you also want them to continue using the app.

字數限制

標題: 30 個字元
主文: 120 個字元
按鈕: 15 個字元

AWhen the error occurs, the Search button is disabled.BAn error message with a countdown timer informs users that the app is trying to reconnect.

溝通策略

  1. The app should not block users from using it or deleting all input data, therefore, using inline notification instead of the dialog componment
  2. Since the action for the user is to wait, the message focuses on keeping the user informed about the current situation
  3. Include a countdown timer in the error message to inform the users that the app is reconnecting to the system.

To create a sense of personalization, the search feature should be disabled initially (fig A).
After that, show the form to greet the user and ask for their "preferred" name and the city where they plan to buy their car.

推銷內容

情境 #01

A user is a working parent, and a big sports fan, in the midst of their favorite sports season who can no longer attend games.

挑戰

Write a promotional screen for an app that lets a user choose teams, sends game reminders, real-time score updates and highlight videos.

字數限制

標題: 40 個字元
主文: 175 個字元
按鈕: 25 個字元

AMake the headline a direct call to action for users to enable notifications.
BTell users the benefits of the notification.
CUse the same keyword "set up" to to reinforce the call to action.
DDepends on the business need, the "not now" button provides an opportunity for the promotional ad to reappear.

溝通策略

  1. To make the message clear and concise, the headline should explicitly tell users what the app wants them to do.
  2. For the call-to-action buttons, use terms that users can predict what will happen
  3. Tell users what's the benefits of the notification

文字的語氣

情境 #02

The user is a casual music fan and (on occasion) goes to live concerts. They have a music player app on their phone.

挑戰

Tell the user that one of their favorite bands is playing live in their town and compel them to want to go.

字數限制

標題: 30 個字元
主文: 45 個字元
按鈕: 25 個字元

AUse the verb "check" instead of "purchase" to alleviate user concerns about immediate charges from the app.
BTo boost users' interest, add a "Start Listening" button for instant access to the band's music.

溝通策略

  1. For Call-to-action buttons, focus on "what's clear" instead of "what's short"
  2. Use words like "exclusive" and "Selling fast" to create a sense of urgency
  3. Venue and date are essential information
  4. If feasible, the promotional page should incorporate in-app features, such as playing a special playlist

文字的語氣

情境 #03

A user is in their favorite supermarket. They open the supermarket’s app on their phone to see what’s on sale and are greeted by a promotion.

挑戰

Write a promotional home screen for a subscription service that delivers groceries to the user once-a-month for a flat fee.

字數限制

標題: 45 個字元
主文: 175 個字元
按鈕: 25 個字元

AThe headline emphasizes the advantages of relieving customers' pain points. BThe headline in Version B explains the subscription model.

溝通策略

  1. Make the layout easy to scan, at because users will be reading the content in a distracting environment
  2. Leverage current news, such as a heat wave, to showcase how the subscription model can tackle customer pain points.
  3. A/B test is needed
  4. Be clear and transparent, help customers to predict what they will get

文字的語氣

情境 #04

An elderly user is doing a Google search to find an easy way to buy contact lenses online.

挑戰

Write a title and meta description for a website that sells subscription contact lenses delivered to a user every 30 days—convince them to try it.

字數限制

標題: 60 個字元
Meta說明文: 160 個字元

APlace the brand name after the keywords "subscription contact lenses"; explain the service at the beginning of the body text.
BOpposite to ver. A, the keywords are placed at the beginning; communicate the benefits at the beginning of the body text.

溝通策略

  1. A/B test is needed
  2. Place keywords and message at the beginning to accommodate F-shaped reading pattern.
  3. Utilize relatable language to explain subscription model.
  4. To set customers' expectations, briefly explain the benefits.
